## Project Proposal Application

**Proposal Title:**
PyArrow Improvemnets for Astronomy

**Organisation:** LINCC frameworks, Apache Arrow

### **Summary:**

*Explain why this project is attractive to you and why you think you can do it.*

This project asks basically for one thing - support for nested type data in `PyArrow`. As nested data forms the core of how astronomy data is stored and processed, this would enable great performance boosts.

I am a geek when it comes to optimizations, my competitive programming was only fueled by one thing - optimization. I take up this project as a challange to provide the most performant upgrades to work with nested data in `PyArrow`!

Also the structure of a really good project like Apache Arrow will and already has tought  me a lot about how projects are structured and managed. I consider it to be a masterclass in structuing C++ project, as there is a beauty in such complexity

### Deliverables

**1.** Multi-threaded Parquet Reader for list of structs.

**2.** Ability to select selectively load column in nested structures like list of structs.

**3.** Compute Kernel Enhancements for Nested Types

**4.** Python API update and docstring update with regards to the above changes

**5.** Tests & Documentation
